# ohno... Another block-style, markdown suppoted, rich editor

Ohno 是一个块状风格（block-style）的富文本编辑器。它的目标是为所有笔记型软件提供一个统一的底层编辑器实现，从而能够在未来搭建起一个足够完善的开源个人笔记体系生态。

ohno 已经实现的特性包括：

- 对中文用户友好的行为定义
- Markdown 风格的编辑体验
- 清晰的历史记录功能
- 简单易扩展的组件/插件系统
- GPL 开源协议
- 序列化支持
- 只会越修越少的 bug
- 可自定义的快捷键系统

## 都 3202 年了，为什么还要造新轮子

我一直想打造一个全方位的个人知识管理系统，而我的预期中，一个完美的管理系统离不开好的记录方式。在我曾经的一个笔记软件的开发尝试中，我调研的绝大多数（精力有限）编辑器，无法同时兼顾以下几个特性：

- 开源（为了开发）
- 较好的用户体验（比如 list 和 paragraph 之间的转换）
- 较稳定的 undo 和 redo（如重复复制粘贴 list 和 paragraph 的内容 10 次并撤销不出错）
- 是富文本编辑器（纯 Markdown 编辑器，如 vscode，在可视化方面终究存在局限性）
- 代码相对容易理解，对社区开发者友好（一些基于 canvas 或者虚拟节点渲染的项目对其他参与进来的开发者而言要了解的内容太多）
- 是我本人写的（雾）

ohno 最初只是一个尝试，是一个乐子，但在开发十几天后，在我对第一个最基本的插入和删除命令反复 undo/redo 没有报错时，ohno 的技术路线就被验证成功了。从此 ohno 就有了让我继续投入，并且对外宣传的意义。
